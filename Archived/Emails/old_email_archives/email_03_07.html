The notions of row-major and column-major are from the standpoint of the person 
writing the algorithm and/or program, and not designing the hardware.

So, I ask you a question: What does the person writing the progam know about the
organization of the memory chips?  Usually: nothing.  At the level of the algorithm
and the program, the two-dimensional array is an abstract concept.  Elements need 
to be stored in some particular sequential order, since memory is sequentally 
addressed.  Column-major order dictates that the address of A[1,0] should be the 
next sequential address after A[0,0].  Whether those two data elements are actually
physically next to each other in memory depends on how the memory chips are actually 
organized by the hardware designer/computer architect..

Bottom line: to the algorithm designer, column-major order simply means the addresses
of A[1,0] and A[0.0] are adjacent.  The hardware designer is free to select address 
bits for each structure in memory according to what appears to him/her to be optimal
from the standpoint of processing.
